# Κληρονομικότητα
## *Πρόοδος*:
-Η κληρονομικότητα δεν είναι μέρος της προόδου.
-Η λογική είναι για να κάνω pass
-Βοηθάει στην προετοιμασία
-Σήμερα ανοίγουν οι αιτήσεις
-Δεν γίνεται αναπλήρωση
## Εργασία 
Παράταση παράδοσης μέχρι και μετά το πάσχα.
## Πρόγραμμα:
-Θα κάνουμε μετα το πάσχα
# **Τι είναι κληρονομικότητα;**
-Δημιουργία ιεραρχικών ταξινομίσεων
-Κληρονομικότητα ιδιοτήτων από ανώτερες σε κατώτερες κλάσεις.
-Αντί να ορισουμε διαφορετικές κλάσεις απλά λέμε ότι οι ιδιότητες κληρονομούνται.
-Μας βοηθάει έτσι να εξοικονομήσουμε χώρο και χρόνο.
*Οι κλάσεις που κληρονομούν τις ιδιότητες της υπερκλασης, έχουν όλες της ιδότητες της υπερκλάσης + τις υπόλοιπες.*
Μπορούμε έτσι να δημιουργήσουμε σχήμα πολυεπίπεδης ιεραρχίας.
## Σύνταξη:
```cpp
class derived_class : public parent_class
{
};
```
Λέμε ότι η νέα κλάση είναι puplic παραγόμενη.
Έτσι θα έχει όλες τις μεθόδους και τις μεταβλητές.
Public -> Public
Private -> Private
Το μόνο που αλλάζει έιναι ο τρόπος προσπέλασης.
Mπορώ να έχω μεθόδους με το ίδιο όνομα στην parent_class και στην derived_class.
e.g. Έχω μία parent_class shapes and the subclass is triangle.
 #**Polymorphism**:
Μπρορώ να έχω το ίδιο όνομα για δύο κλάσεις(την parent_class και την subclass)
*Πως διαφοροποιούμαι?*
-Με scope resolution.
i.e. Έχω Parent class book and subclass ebook
ebook is public derived from book.
Both classes have the same method info.
Αν τις καλέσω απευθείας πχ.
```cpp
Book B1;
Ebook B2;
//call:
B1.info(); //Book info
B2.info(); //Ebook info
B2.Book::info();//Book info
```
## Access Control:
### Protected:
Intermediate access.
Το protected μου δίνει την ευκαιρία να δω την πληροφορία.
Private: -Μόνο τα μέλη της κλάσης.
Protected: -Πρόσβαση και από παράγωγη κλάση όχι από εξωτερικό κώδικα.
Public: -All public
Όταν υπάρχει κληρονομικότητα πρέπει να σκέφτομαι και protected.
Some methods that are relevant to classes may go into protected
## Τρόποι δημιουργίας παράγωγων κλάσεων(διαφοροποίηση ανάλογα με τον ορισμό)
Έχω τη δυνατότητα να είναι:
-*Public:*
  Public -> Public
  Protected -> Protected
  Private -> Δεν κληρονομούνται!!!!
-*Private:*
  Private -> Not inherited
  Public -> Private
  Protected -> Private
Πρέπει να κάνω άλλες μεθόδους για να έχω πρόσβαση.
Η πληροφορία της βασικής μένει στον κύκλο της κληρονομικότητας.
Άρα η parent_class δεν παρέχει πρόσβαση στην πληροφορία.
-*Protected:*
  Private -> Private
  Pubic -> Protected
  Protected -> Protected
**Προσοχή!**
Στην κληρονομικότητα πρώτα δεσμεύεται μνήμη για την υπερκλάση και μετά για την παραγόμενη.
Η σειρά καταστοφής είναι η ανάποδη.
